# CareerSignals v1.0.0

<!-- markdownlint-disable MD013 -->

CareerSignals v1.0.0 is the first stable release of the hosted, multi-user job-search intelligence platform. It turns a shared, continuously refreshed job market into a private and explainable action queue for each user.

**Live product:** [jobs.swiftaihub.com/careersignals](https://jobs.swiftaihub.com/careersignals)

## Highlights

- Personalized job matching with explainable scores and five match tiers.
- A complete workflow for job discovery, filtering, prioritization, and application tracking.
- Dashboard analytics for market volume, match quality, visa signals, work arrangements, and application progress.
- Guided search preferences, skill profiles, match-priority controls, and revision history.
- Separate platform-owned and user-owned dbt pipelines with tenant-safe, atomic publication.
- A read-only, password-free Demo workspace with 20 curated jobs.
- Full account, entitlement, administration, audit, and password-recovery workflows.
- Scheduled multi-source job collection and production-ready deployment automation.

## User experience

### Public product site

- Responsive product landing page with feature explanations and a four-step product walkthrough.
- Direct registration, login, pricing, and live Demo entry points.
- Product illustrations, interactive previews, mounted-path navigation, and accessible loading and error states.
- Search metadata, Open Graph and Twitter cards, sitemap, robots policy, web manifest, and application icon.
- Production support for hosting the entire web application below `/careersignals`.

### Authentication and account lifecycle

- Sign in with either username or email through Supabase Auth.
- New registrations enter a pending state until an administrator activates the account.
- Activation starts the user's time-limited entitlement instead of consuming access while approval is pending.
- Dedicated experiences for pending, active, expired, suspended, and deleted accounts.
- Secure logout with mounted-path cookie cleanup and cross-origin request protection.
- Email-based forgot-password and reset-password flows.
- Authenticated password changes and support for administrator-required password changes.
- Server-side session validation across public, protected, and Admin navigation.

### Read-only Demo

- One-click Demo access without a password.
- Short-lived, server-signed Demo sessions.
- A deterministic tenant containing exactly 20 curated job matches.
- Access to the Dashboard, Jobs, Top Matches, Skill Gap, Companies, and Settings previews.
- Database and API protections that prevent Demo configuration changes, status changes, pipeline runs, and full exports.

## Personalized job intelligence

### Explainable matching

- User-specific scoring across title alignment, required skills, industry, salary, work arrangement, and visa signals.
- Location and employment preference filtering before scoring.
- User-adjustable ranking weights that always total 100%.
- Five result tiers: Excellent Match, Strong Match, Good Match, Possible Match, and Low Priority.
- Human-readable reasoning and component scores for every ranked opportunity.
- Skill alias expansion so safe variations of the same capability match consistently.
- Candidate-location matching that handles remote roles, city/state variations, metro areas, multi-location postings, and country-level preferences.

### Dashboard

- Summary metrics for total matches, Excellent and Strong matches, remote or hybrid roles, visa-signal coverage, and personal-result freshness.
- Job Search Funnel showing the progression from the available job universe to applications, interviews, and offers.
- Job Volume Over Time comparing global jobs, personal matches, and application activity.
- Distribution charts for category, match tier, work arrangement, and visa signal.
- Latest Top Matches preview with score, salary, location, and visa context.
- Real zero, missing, and partial-data handling instead of fabricated analytics values.

### Job Explorer

- Paginated, sortable job table backed by user-scoped serving data.
- Full-text search across relevant job fields.
- Filters for category, match-score range, company, industry, location, location group, work arrangement, visa signal, application status, and posting date.
- Dynamic filter options and count-aware facets derived from the current dataset.
- Normalized location groupings for remote, domestic, international, and unclassified jobs.
- Detailed role view with score breakdown, job metadata, salary, visa evidence, source, and application link.
- Sort controls for match, salary, date, company, location, and other supported job fields.

### Application tracking

- Statuses for Not Applied, Saved, Applied, Interview, Rejected, Offer, and Archived.
- Optional notes stored with each status update.
- Status history used by the dashboard funnel and time-series analytics.
- User-specific status records that never change another user's view of the same shared job.

### Top Matches

- A focused review queue for the highest-scoring opportunities.
- Match tier, score, salary, location, company, visa evidence, and application state at a glance.
- Direct actions to save, apply, or archive roles from the prioritized workflow.

### Skill Gap

- Market-demand analysis across the user's target roles.
- Separation of skills already covered by the user's profile from missing or underrepresented skills.
- Job counts, demand frequency, skill groups, and example matching titles.
- Sorting and filtering for prioritizing the most useful development opportunities.

### Company Priority

- Employer-level rollups across matching-role count, average and highest score, salary midpoint, and visa signals.
- Best matching role for each company.
- Priority tiers that surface companies with the strongest concentration of high-quality opportunities.
- Sortable company analysis for deciding where to focus networking and applications.

### Excel export

- Private, user-scoped `.xlsx` export generated on demand.
- Separate worksheets for My Matches, Category Summary, Skill Gap, and Company Priority.
- Collision-safe, date-stamped filenames without tenant identifiers.
- Up to 5,000 ranked matches per export.
- Demo exports are disabled.

## Personalization and settings

### Guided search preferences

- Target roles with automatically generated search-title expansions.
- Suggested and custom industries.
- Searchable country, city, state, region, metro-area, and country-level locations.
- Remote, hybrid, and on-site work arrangements.
- Multiple employment types.
- Visa and work-authorization preferences without requiring users to maintain keyword lists.
- Preferred and excluded companies.
- Excluded job titles.
- Optional minimum and preferred compensation.
- Dynamic suggestions from the shared job universe for locations, industries, companies, and job titles.

### Skills and experience

- Profession-agnostic skill entry for tools, domains, certifications, and areas of expertise.
- Optional user-defined skill categories.
- Automatic alias generation and reuse of the shared alias catalog.
- Deduplication and normalization of skill names and aliases.
- Generated-data preview before preferences are used by the matching pipeline.

### Match priorities

- Interactive priority controls for every weighted scoring dimension.
- Radar visualization of the current priority shape.
- Automatic redistribution in whole percentages so the total remains 100%.
- Lockable dimensions for preserving selected values while adjusting the rest.
- Strict server-side validation for totals and value ranges.

### Configuration history

- Atomic configuration bundles covering the candidate profile, job preferences, and skill taxonomy.
- Immutable revision history with timestamps and bundle identity.
- Coherent restore points that restore all related settings together.
- Restore-as-new-revision behavior so history is never rewritten.
- Full reset and targeted field-reset actions.
- Optimistic concurrency checks that reject stale updates instead of silently overwriting newer settings.

### Personal refresh controls

- One-click refresh using the latest saved settings and skills.
- Current daily quota, successful usage, remaining runs, and reset time.
- Production limit of four successful personal refreshes per user per quota window.
- Quota windows reset at 6:00 AM in `America/New_York`.
- Failed and cancelled runs do not consume the successful-run quota.
- Run timeline with queued, waiting, running, completed, failed, and cancelled states.
- User cancellation for eligible queued runs.
- Shared-data freshness and next scheduled refresh displayed separately from personal-result freshness.

## Job acquisition and processing

### Connector framework

- Connector implementations for Adzuna, SerpAPI, Greenhouse, Lever, USAJobs, and deterministic mock data.
- Runtime source enablement so deployments can activate only configured and credentialed providers.
- Per-source page and query budgets.
- Configurable HTTP timeouts, retries, and backoff.
- Concurrent source collection with bounded production concurrency.
- Safe error sanitization so provider responses and credentials are not exposed through user messages or persisted diagnostics.

### Shared market refresh

- System-owned global refreshes that combine acquisition needs across eligible active users.
- Query normalization and deduplication to avoid repeating equivalent external searches.
- Configurable freshness filtering before jobs enter shared raw storage.
- Default scheduled refreshes at 7:00 AM, 4:00 PM, and 9:00 PM Eastern time.
- Manual global refreshes from the Admin console and trusted CLI.
- Overlap protection so only one global refresh owns the publication path.
- Per-source run metadata, success and failure tracking, included-user counts, and published snapshot identity.
- First-user bootstrap orchestration is available for non-production deployments through an explicit trigger mode.

### Normalization and enrichment

- Stable job identifiers and cross-source deduplication.
- Job-title, company, industry, seniority, employment-type, and work-arrangement normalization.
- Location normalization and location-group facets.
- Salary range parsing, including annualization of hourly compensation.
- Protection against non-finite salary values and invalid timestamps during publication.
- Visa sponsorship classification with Positive, Negative, and Unknown outcomes, supporting evidence, and confidence.
- Restrictive visa language takes precedence when descriptions contain contradictory signals.
- Skill extraction from job descriptions using the user's skills and aliases.

## Data platform and dbt

### Two deliberately separate selectors

- `shared_refresh` builds only candidate-independent shared resources.
- `user_refresh` builds only one trusted user and run partition.
- Selectors are fixed server-side and cannot be supplied by a browser request.
- Every user model requires validated `user_uuid` and `run_uuid` context.
- `--full-refresh`, arbitrary selectors, and unqualified multi-user deletion are rejected by the supported runners.

### Shared models

- Staging of processed connector jobs.
- Deduplication into the canonical shared job universe.
- Canonical job mart for serving and relationship validation.
- Per-source freshness mart with active-job counts and connector-run identity.

### User models and marts

- User-scoped staging for job preferences, candidate skills, skill aliases, and application status.
- Candidate-job filtering before expensive skill expansion and scoring.
- Skill-level matching and score rollups.
- Final scored-jobs mart with component scores, match tier, visa evidence, salary, and application state.
- Dedicated marts for Top Matches, Category Summary, Skill Gap, and Company Priority.
- Compatibility models for existing candidate-skill and exploded-skill consumers.

### Safe publication

- Incremental `delete+insert` models with schema synchronization.
- Exact user/run partition cleanup hooks instead of global truncation.
- Atomic publication of shared snapshots and personal results into PostgreSQL serving tables.
- Cumulative personal-result handling that updates touched jobs while preserving unaffected current rows.
- A failed build or publication leaves the previously successful personal result set available.
- NaN, infinity, out-of-range timestamps, and cross-user result rows are rejected or safely normalized before serving publication.

## Security and isolation

- Supabase JWT verification with issuer, audience, and JWKS validation.
- Server-only service-role credentials; browser requests go through an allowlisted same-origin Next.js BFF.
- Cross-origin mutation and logout rejection.
- FastAPI authorization checks for active accounts, non-Demo mutations, and Admin-only operations.
- Row Level Security enabled and forced on tenant, analytics, pipeline, billing, entitlement, and audit tables.
- Authenticated identity is derived from the verified session, never a client-supplied user UUID.
- Shared connector acquisition cannot import or execute from the personal pipeline.
- Demo write protection is enforced at both the API and database-trigger layers.
- Current-result triggers prevent stale or foreign run data from becoming active.
- Internal error details remain available to trusted operators while user-facing messages stay sanitized.
- Exact production CORS, referrer, frame, content-type, and permissions policies.

## API and compatibility

- FastAPI endpoints for authentication, current-user context, jobs, facets, dashboard analytics, top matches, category summaries, skill gaps, company priorities, status updates, exports, settings, preferences, pipeline runs, quotas, freshness, and Admin operations.
- Server-side pagination and validated sorting for large job datasets.
- Same-origin frontend proxy with a narrow backend path allowlist.
- Health endpoints for deployment and service monitoring.
- Legacy `/api/pipeline/run`, `/api/dbt/run`, `/api/dbt/test`, and legacy Excel paths return `410 Gone` instead of exposing unsafe execution surfaces.

## Requirements

- Python 3.11 or 3.12.
- Node.js 22 or newer.
- Supabase CLI and a local or hosted Supabase project.
- A MotherDuck database and token for production-style analytics execution.
- Credentials for any live connector sources enabled by the deployment.

Apply all migrations in `supabase/migrations/` in numeric order before starting v1.0.0. Then bootstrap the first Admin and seed the Demo tenant as described in the [README](../../README.md).

## Known limitations

- Billing remains a manual entitlement workflow. The pricing page and estimated MRR are product placeholders; there is no Stripe checkout or webhook integration in v1.0.0.
- The repository enables only the deterministic mock connector by default. Live providers require explicit source configuration and credentials.
- Production forces scheduled shared refreshes and disables first-user bootstrap. A new user matches against the latest successfully published shared snapshot.
- Production personal matching is intentionally limited to one concurrent MotherDuck writer.
- The production API enforces four successful personal refreshes per user per daily quota window.
- Supabase and MotherDuck are external runtime dependencies and are not emulated by the root Docker Compose stack.

## Breaking changes

This is the initial stable release, so there is no previous stable version to migrate from. Unsafe legacy pipeline and dbt execution endpoints are intentionally unavailable and return `410 Gone`.
